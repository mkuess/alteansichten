1. Context
This is a Laravel/Filament project running on Replit.

There is a recurring URL/port problem with the admin backend.

2. Problem
When I open /admin directly, the page does not load correctly.
I have to manually add :8000 to the URL.

Once I am inside the backend, internal links seem to work because they include :8000.

This is not acceptable. The admin backend should work from the normal Replit preview URL without manually adding :8000.

3. Goal
Fix the Laravel/Replit URL and port configuration so /admin loads correctly without manually adding :8000.

4. Requirements
- Opening /admin from the normal Replit preview URL must work.
- Do not require manually adding :8000.
- Internal Filament links should use the correct public Replit URL.
- Asset URLs should load correctly.
- Livewire/Filament requests should use the correct URL.
- Do not hardcode a temporary preview domain if avoidable.
- Do not change admin UI design.
- Do not change forms.
- Do not change map functionality.
- Do not change upload logic.
- Do not change database schema.
- Do not refactor unrelated files.

5. Technical checks
Please inspect and fix the relevant configuration:

- .env
- APP_URL
- Filament panel/provider configuration
- Vite/dev server configuration if relevant
- Laravel trusted proxies / trusted hosts if relevant
- Replit port/proxy settings
- any custom URL generation logic
- any hardcoded :8000 links
- any hardcoded absolute admin URLs

6. Important direction
The fix should make Laravel generate URLs correctly for Replit’s public preview/proxy environment.

If APP_URL is currently set to a URL with :8000, check whether that is causing incorrect behavior.

If APP_URL is missing or wrong, update it appropriately for Replit.

If Replit requires the Laravel server to listen on 0.0.0.0:8000 internally, keep that internal server behavior, but do not force the public browser URL to require :8000.

7. Safety
Before changing anything:
- identify exactly why /admin does not load without :8000
- identify whether the issue is caused by APP_URL, proxy headers, Filament URL generation, assets, or hardcoded links

Only change the minimum necessary configuration/code.

Do not touch:
- PlaceResource
- MediaItemResource
- upload fields
- map page behavior
- geocoding
- database migrations
- admin styling/theme
- dashboard widgets

8. Expected result
After the fix:
- I can open /admin directly from the normal Replit preview URL
- the backend loads correctly
- internal Filament links work
- assets load correctly
- no manual :8000 in the browser URL is needed

9. Stop condition
After fixing, stop and summarize:
- what caused the wrong :8000 behavior
- which files/settings were changed
- what APP_URL is now set to
- whether any hardcoded :8000 links were removed
- confirmation that no unrelated resources, forms, map, upload logic, theme, or database schema were changed

Please stop afterward and do not implement any further steps without being asked.
