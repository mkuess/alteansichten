1. Project context

This project is called “Alte Ansichten”.
It is a Laravel 11 and Filament 3 project for a historical archive platform.

The first MVP needs places / locations as the core content entity.
Places will later belong to municipalities and categories, have public profile pages, media, QR codes, submissions, and map coordinates.

This task only creates the places table and the Place model.

Replit is only the development environment.
The project must remain standalone and deployable later on my own Linux VM at spaceship.com.

6. Expected files / changes

Expected files:

Expected places table fields:

- id
- municipality_id nullable foreign key referencing municipalities.id
- category_id nullable foreign key referencing categories.id
- title
- slug
- summary nullable text
- story nullable long text
- street nullable
- house_number nullable
- postal_code nullable, max length 20
- address_text nullable
- latitude nullable decimal, precision suitable for coordinates
- longitude nullable decimal, precision suitable for coordinates
- location_precision nullable string
- location_note nullable
- period_from nullable unsigned small integer
- period_to nullable unsigned small integer
- approximate_date_text nullable
- status string, default draft
- visibility string, default public
- internal_reference_code nullable, max length 50
- timestamps
- soft deletes

Expected status values for future use:

- draft

- archived

Expected visibility values for future use:

- public
- unlisted
- private

For now, keep status and visibility as simple string fields, not PHP enums.

The model should use SoftDeletes and allow mass assignment for the relevant fields.
