1. Project context

This project is called “Alte Ansichten”.
It is a Laravel 11 and Filament 3 project for a historical archive platform.

The current priority is to build the backend first so entries can be maintained properly in Filament.
The public frontend will be handled later.

Media items already exist.
Now the project needs media_links so one media item can be connected to places and municipalities.

For the first MVP, media_links should only support:
- place
- municipality

People and organizations are not part of the first MVP yet.

The Claude backend UI mockup is saved as a later design reference.
Do not apply the custom design yet.

Replit is only the development environment.
The project must remain standalone and deployable later on my own Linux VM at spaceship.com.

6. Expected files / changes

Expected files:

- database/migrations/xxxx_xx_xx_xxxxxx_create_media_links_table.php
- app/Models/MediaLink.php
- app/Models/MediaItem.php updated with mediaLinks relationship

Expected media_links table fields:

- id
- media_item_id foreign key referencing media_items.id
- linkable_type string
- linkable_id unsigned big integer
- relationship_type nullable string
- is_primary boolean, default false
- sort_order unsigned integer, default 0
- period_from nullable unsigned small integer
- period_to nullable unsigned small integer
- timestamps

Recommended indexes:

- media_item_id
- linkable_type + linkable_id
- media_item_id + is_primary

Expected future linkable_type values for MVP:

- place
- municipality

Expected future relationship_type examples:

- main_subject
- panorama_of
- shows_detail
- related
- document_about
- source_for

For now, keep these as simple string fields, not PHP enums.

The model should allow mass assignment for the relevant fields.
