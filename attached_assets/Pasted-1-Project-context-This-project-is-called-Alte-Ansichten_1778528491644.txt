1. Project context

This project is called “Alte Ansichten”.
It is a Laravel 11 and Filament 3 project for a historical archive platform.

The current priority is to build the backend functionality first so entries can be maintained properly in Filament.
Design refinement and public frontend work will be handled later.

QR code records already exist and can be managed in Filament.
Now the backend needs a simple service/action to create a QR code record for a place.

This task should only create the QR code record and target URL.
It should not generate actual QR image files yet.

3. Exact task

Add a simple backend action to create a QR code record for a place from the PlaceResource.

Do not generate QR image files yet.

4. What Replit should do

- Add a small service class or helper method for creating QR code records for places.
- The generated QR code record should include:
  - place_id
  - code
  - target_url
  - scan_count = 0
- Generate a stable unique code based on the place slug plus a short unique suffix if needed.
- The target_url should point to the future public place URL, for example:
  - /orte/{place-slug}
- Add an action to PlaceResource, preferably on the edit page or table row:
  - “QR-Code erstellen”
- The action should:
  - check whether the place already has a QR code
  - create one only if none exists
  - show a success notification
  - not create duplicates
- Keep this standard Filament and Laravel.
- Verify that the action can create a QR code record for an existing place.
