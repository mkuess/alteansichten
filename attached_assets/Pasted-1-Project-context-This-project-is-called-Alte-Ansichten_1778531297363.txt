1. Project context

This project is called “Alte Ansichten”.
It is a Laravel 11 and Filament 3 project running in Replit.

The Laravel app itself works on port 8000.
The Filament backend is reachable and styled.
However, navigation inside the Replit embedded Preview panel still fails.

The visible Replit message is:

“The URL bar is not supported when navigating to external pages.
You can open in a new tab to use your browser instead.”

This happens when clicking Filament sidebar navigation links.

2. Current state

The admin panel loads.
CSS and JS assets now return HTTP 200.
Filament styling is working again.
The Laravel server is running.
Tests pass.

But in the embedded Replit Preview, clicking Filament sidebar links causes Replit’s preview frame to treat the navigation as an external page.

This is likely caused by absolute URLs generated for Filament navigation, APP_URL / ASSET_URL configuration, or a Replit Preview limitation.

3. Exact task

Diagnose and fix Filament admin navigation so internal admin links work correctly inside Replit Preview if possible.

If this is a Replit Preview limitation that cannot be fixed safely in Laravel, clearly report that and confirm the correct workaround is to open the app in an external browser tab.

  - relative paths like /admin/places
