1. Project context

This project is called “Alte Ansichten”.
It is a Laravel 11 / Filament 3 project running in Replit.

The correct Replit development URL must include port 8000:

https://...janeway.replit.dev:8000

The admin panel works when :8000 is manually added to URLs.

However, Filament-generated sidebar/menu links still appear without :8000. When those links are clicked, the browser opens the portless URL and Replit returns 502.

2. Current state

APP_URL was intended to be set to:

https://...janeway.replit.dev:8000

ASSET_URL was intended to be set to:

https://...janeway.replit.dev:8000

But in the actual browser, generated admin links still appear as:

https://...janeway.replit.dev/admin/categories

instead of:

https://...janeway.replit.dev:8000/admin/categories

When I manually add :8000, the page works.

3. Exact task

Fix Filament/Laravel URL generation so all admin navigation links include :8000 in Replit.

- Check the actual href values of sidebar/menu links.

route('filament.admin.resources.categories.index');

  APP_URL has :8000
