4. What Replit should do

First, reproduce the upload failure in the browser:

- Open /admin/media-items/create
- Try uploading a very small JPG under 500 KB

Then inspect:

- Browser Network request for the failed upload
- exact request URL
- exact HTTP status code
- response body
- Laravel log after the upload attempt
- Replit server console after the upload attempt

Also check:

- whether the upload request URL is same-origin with the admin page
- whether the upload request includes the expected port/origin currently used by the working admin UI
- whether the request fails before Laravel or inside Laravel

Run these checks:

php artisan test

Check PHP/Laravel values:

php -r "echo 'upload_max_filesize=' . ini_get('upload_max_filesize') . PHP_EOL; echo 'post_max_size=' . ini_get('post_max_size') . PHP_EOL; echo 'memory_limit=' . ini_get('memory_limit') . PHP_EOL;"

Check directories:

- storage/app
- storage/app/public
- storage/app/public/media-items
- storage/app/livewire-tmp
- storage/framework/cache
- storage/framework/sessions
- storage/framework/views

Make sure Laravel can write to:

- storage/app/livewire-tmp
- storage/app/public/media-items

If a directory is missing, create it.
If permissions are wrong, fix permissions.

- If the upload fails because storage/app/livewire-tmp is missing, create it.

- If the upload fails because storage permissions are wrong, fix permissions.
- If the upload fails because PHP upload_max_filesize/post_max_size is too low, add the minimal local PHP configuration needed for the Replit dev server and explain how it works.
