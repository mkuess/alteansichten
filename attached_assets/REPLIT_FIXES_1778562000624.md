## 1. Bug: Server Error auf Dashboard (TopMunicipalitiesWidget)

Im Widget `app/Filament/Widgets/TopMunicipalitiesWidget.php` fehlt die Variable `$municipalities` im View. Fix:

```php
// app/Filament/Widgets/TopMunicipalitiesWidget.php
namespace App\Filament\Widgets;

use App\Models\Municipality;
use Filament\Widgets\Widget;

class TopMunicipalitiesWidget extends Widget
{
    protected static string $view = 'filament.widgets.top-municipalities-widget';
    protected int | string | array $columnSpan = 'full';
    protected static ?int $sort = 3;

    public function getHeading(): string { return 'Top-Gemeinden'; }

    protected function getViewData(): array
    {
        $municipalities = Municipality::withCount('places')
            ->orderByDesc('places_count')
            ->limit(8)
            ->get();
        $max = $municipalities->max('places_count') ?? 1;
        return compact('municipalities', 'max');
    }
}
```

```blade
{{-- resources/views/filament/widgets/top-municipalities-widget.blade.php --}}
<x-filament-widgets::widget>
    <x-filament::section :heading="$this->getHeading()">
        @if($municipalities->isEmpty())
            <p class="text-sm" style="color: var(--ink-3)">Noch keine Gemeinden angelegt.</p>
        @else
            <ul class="space-y-3">
                @foreach($municipalities as $m)
                    @php $pct = $max > 0 ? round(($m->places_count / $max) * 100) : 0; @endphp
                    <li class="flex items-center gap-3">
                        <div class="w-8 h-8 rounded-md flex items-center justify-center font-serif text-xs"
                             style="border:1px solid var(--line); background: var(--bg)">
                            {{ strtoupper(substr($m->name, 0, 2)) }}
                        </div>
                        <div class="flex-1 min-w-0">
                            <div class="text-sm font-medium truncate">{{ $m->name }}</div>
                            <div class="h-1 rounded-full mt-1" style="background: var(--line-2)">
                                <div class="h-full rounded-full" style="width: {{ $pct }}%; background: var(--accent)"></div>
                            </div>
                        </div>
                        <span class="text-xs font-mono" style="color: var(--ink-3)">+{{ $m->places_count }}</span>
                    </li>
                @endforeach
            </ul>
        @endif
    </x-filament::section>
</x-filament-widgets::widget>
```

Beachte: `Municipality` muss `places()` HasMany haben.

## 2. Theme greift nur teilweise — vollständige Übernahme erzwingen

Wenn Tailwind/Filament-Theme nicht vollständig durchschlägt, liegt es fast immer an:

**A)** Theme-CSS wird nicht kompiliert. Prüfe:
```bash
ls -la public/build/
npm run build
php artisan filament:assets
php artisan optimize:clear
```

**B)** Panel-Provider lädt das Theme nicht. In `app/Providers/Filament/AdminPanelProvider.php` muss stehen:
```php
->viteTheme('resources/css/filament/admin/theme.css')
```

**C)** Vite-Konfiguration fehlt. In `vite.config.js`:
```js
input: [
    'resources/css/app.css',
    'resources/js/app.js',
    'resources/css/filament/admin/theme.css',  // ← muss da sein
],
```

**D)** Tailwind-Config des Admin-Themes. In `resources/css/filament/admin/tailwind.config.js`:
```js
import preset from '../../../../vendor/filament/filament/tailwind.config.preset'

export default {
    presets: [preset],
    content: [
        './app/Filament/**/*.php',
        './resources/views/filament/**/*.blade.php',
        './vendor/filament/**/*.blade.php',
    ],
}
```
