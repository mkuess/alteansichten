## PROMPT FÜR DEN REPLIT-AGENTEN

Erstelle ein neues Laravel 11 Projekt mit Filament 3 Admin Panel. Der Mockup im Anhang (`Alte Ansichten Backend.html`) zeigt das gewünschte Endergebnis — du musst Filament so customizen, dass das Backend **exakt so aussieht**.

### 1. Projekt-Setup

```bash
composer create-project laravel/laravel . "^11.0"
composer require filament/filament:"^3.2"
php artisan filament:install --panels
php artisan make:filament-user
```

Panel-ID: `admin`, Pfad: `/admin`.

**Designprinzipien:**
- Warmer, papierartiger Hintergrund (NICHT weißes Tailwind-Standard-Grau)
- Karten: weiß, Border `1px solid var(--line)`, Radius 12 px, KEIN harter Shadow — höchstens `0 1px 0 rgba(0,0,0,.02)`
- Überschriften IMMER in Newsreader (serif), niemals fett-sans
- Tabellen mit großzügigem Padding (px-5 py-3.5), Header `text-[11px] uppercase tracking-wider`
- Status-Pills: small caps, mono, abgerundet (999 px), pastellig (sage/amber/rose/slate)
- Sidebar-Navigation gruppiert mit Mini-Überschriften `text-[11px] uppercase tracking-wider`
- Aktiver Nav-Eintrag: Hintergrund `--accent-bg`, Akzent-Icon `--accent`
- Icons: dünn (stroke 1.5), nicht gefüllt
- Mono-Font für alle Referenzen wie `PLACE-000124`, Lat/Lng, Slugs, Spaltennamen

### 4. Datenmodell — 21 Tabellen aus dem Konzept

Lege per `php artisan make:model -mfsr` für jedes Modell Migration + Factory + Seeder + Filament Resource an:

**Regionen:** `Country`, `State`, `District`, `Municipality`
**Inhalte:** `Place`, `Category`, `MediaItem`, `MediaLink` (polymorph), `Person`, `Organization`, `OrganizationArea`, `PersonPlace`
**Moderation:** `Submission`, `SubmissionMedia`, `Report`
**System:** `User`, `UserRegionPermission`, `QrCode`, `Timeline`, `MaintenanceFlag`, `AuditLog`

Wichtige Felder (auszugsweise):
- `places`: `municipality_id`, `category_id`, `title`, `slug`, `story`, `year_from`, `year_to`, `approximate_date_text`, `lat`, `lng`, `location_status` (enum: exact_place, approximate_area, municipality_only, unknown, multiple_places, not_location_based), `status` (draft, pending, published, archived, rejected), `visibility` (public, unlisted, private), `primary_media_item_id`
- `media_items`: `type` (image, document, newspaper_article, external_link), `year`, `rights_status` (public_domain, with_permission, cited, unknown), `primary_place_id`
- `submissions`: `user_id`, `submission_type` (add_media, add_place, edit_place, report_issue), `target_type`, `target_id`, `rights_confirmed_at`, `terms_accepted_at`, `privacy_accepted_at`, `status`

### 6. Wizard für Eintrag erstellen

`app/Filament/Pages/CreateEntryWizard.php` — `Filament\Pages\Page` mit `Filament\Forms\Components\Wizard`:

Steps (exakt wie im Mockup):
1. **Gemeinde** — Suche/Auswahl aus `Municipality`
2. **Grunddaten** — title, category, story, year_from/to, status, visibility
3. **Adresse & Karte** — street, no, plz, lat, lng (Leaflet via filament-plugin), location_status
4. **Medien** — Drag-and-Drop Upload, pro File: title, year, source, rights_status
5. **Personen & Organisationen** — Repeater mit Beziehungstyp (lived_here, operated_business_here, ...)
6. **Vorschau** — Read-only Zusammenfassung + Veröffentlichungs-Checkliste

Wichtig: **kein** "Bisherige Eingaben"-Sidebar, voller Fokus auf den aktuellen Schritt.

### 7. Dashboard

Widgets in folgender Reihenfolge:
1. **StatsOverviewWidget** — 4 Stats: Standorte (428), Medien (1247), Einreichungen offen (12), Aktive Nutzer (8) — jede mit Delta `+X / 7T`
2. **LatestSubmissionsWidget** — Tabelle der 5 neuesten Einreichungen
3. **TopMunicipalitiesWidget** — Liste mit Bar-Indikator
4. **MaintenanceAlertsWidget** — Frühwarnungen (Medien ohne Zuordnung etc.)

### 8. Reihenfolge für den Agenten

1. Laravel + Filament installieren, Theme-CSS oben einbauen → Login-Screen sollte schon im Stil sein.
2. Migrationen für alle 21 Tabellen + Seeder mit Beispieldaten (Mödling, Sankt Pölten, Berndorf, Stainz, Pfaffenhofen…).
3. Resources der Reihe nach: Place → Municipality → MediaItem → Person → Organization → Submission → Report → User → QrCode.
4. Wizard-Page als letzte.
5. Dashboard-Widgets.
6. Mit `npm run build` Theme finalisieren.

### 9. Akzeptanzkriterium

Vergleiche den fertigen `/admin` Screen mit dem mitgelieferten `Alte Ansichten Backend.html`. Sidebar-Layout, Schriftarten, Farben, Pill-Stile und Tabellen-Optik müssen visuell deckungsgleich sein.
